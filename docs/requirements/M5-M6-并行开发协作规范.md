# M5 / M6 并行开发协作规范

> 日期：2026-06-08  
> 适用范围：两名开发者同时推进 **M5（AI 对话 + Agent）** 与 **M6（素材管理 + 预设）** 时必读。

---

## 1. 结论

**可以并行开发**，但必须遵守本文的模块归属、IPC 契约与合并节奏，避免在 `timelineStore`、`asset-service`、布局壳层上产生不可合并冲突。

| 维度 | 约定 |
|------|------|
| 开发者 A | M5：对话 UI、LLM/Agent、Python 侧（若启用） |
| 开发者 B | M6：素材导入/删除、素材库 UI、预设、SQLite 素材表 |
| 共享层 | `timelineStore` mutations、`packages/shared` 类型、IPC 错误码 |
| 集成周 | 并行第 3–4 周联调；每周至少一次从 `main` 合并 |

详细分工见：[M5-AI对话与Agent-开发规范.md](M5-AI对话与Agent-开发规范.md)、[M6-素材管理与预设-开发规范.md](M6-素材管理与预设-开发规范.md)。

## 4. IPC 与类型契约

### 4.1 素材（M6 实现，M5 消费）

M5 Agent 工具 `importAsset` **不得**自行复制文件，必须调用：

| Channel | 说明 |
|---------|------|
| `main:asset:list` | 列出当前项目素材 |
| `main:asset:import` | `{ filePaths, subprojectPath?, fps? }` |
| `main:asset:pickAndImport` | 系统文件选择后导入 |
| `main:asset:delete` | **M6 实现** `{ assetId, mode?: 'soft' \| 'removeClips' }` |

**`ProjectAsset` 最小字段（M6 保证，M5 依赖）：**

```ts
{
  id: string;
  type: 'image' | 'video' | 'audio';
  publicPath: string;      // 如 /assets/video/{uuid}.mp4
  path: string;            // 项目内相对路径 assets/video/...
  durationInFrames?: number;
  width?: number;
  height?: number;
}
```

导入成功后 M5 可用返回的 `id` + `publicPath` 调用 `createClip`（`source.kind: 'asset'`）。

### 4.2 时间线（M5 / M6 共用）

| 操作 | 唯一入口 |
|------|----------|
| 创建/移动/删除片段 | `timelineStore` → `lib/timeline/mutations.ts` |
| 拖入素材落点 | `placeAssetOnTimeline`（M6）；M5 Agent 等价调用 store 方法 |
| 触发 Remotion 生成 | `timelineStore.scheduleGenerate` / `runGenerate` |
| 延长成片时长 | **M6**（`M6-05`）；M5 不单独实现 |

### 4.3 对话（M5 独占）

| Channel（规划） | 说明 |
|-----------------|------|
| `main:chat:send` | 发送消息，流式响应 |
| `main:chat:history` | 读写 `conversation.json` |
| `main:agent:runTool` | 内部工具链（可选，不暴露给渲染进程裸调） |

## 5. 不建议并行的任务

| 任务 | 原因 | 建议 |
|------|------|------|
| M6-08 SQLite 大迁移 + M5 `importAsset` 联调 | 素材 ID/存储结构未定时 Agent 易返工 | M6 先定 schema 或保持 manifest 双写一期 |
| 两人同时大改 `timelineStore.ts` | 高冲突 | 按 mutation 拆 PR，每日合并 |
| M6-05 延长成片 + M5 `updateClip` 改时长 | 业务规则可能不一致 | 时长上限与弹窗文案由 M6 文档为准 |
| 同时改 `AppLayout.tsx` 分栏比例 | UI 冲突 | 只改各自 Panel 子组件 |

---

## 6. 分支与合并

```
main
 ├── feat/m5-agent    # 开发者 A
 └── feat/m6-assets   # 开发者 B
```

- 每周至少一次将 `main` 合并进各自分支
- 共享文件冲突：**以已合并到 main 的 IPC 契约为准**
- PR 标题建议：`feat(m5): ...` / `feat(m6): ...`
- 联调 PR：`feat(m5+m6): integrate agent importAsset` 在双方核心 PR 合并后进行
