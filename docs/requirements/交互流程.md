# 交互流程

## 核心设计理念
- **简化流程**：解决剪辑师不会用 AE、不会写代码控制 Remotion 的痛点
- **多模态输入**：支持图片参考，一张排版参考图比大量文字描述更准确
- **对话驱动**：通过自然语言 + 参考素材快速生成动画

## 新建项目
- 选择分辨率、帧率、时长，或从模板开始

## 核心工作流（参考文稿 + 参考排版 → 动画生成）
1. **输入参考文稿**：用户提供文案、脚本、数据等内容
2. **上传参考排版**：用户提供参考动画排版的截图/图片（多模态模型解析视觉布局）
3. **Agent 生成**：
   - 多模态 LLM 解析参考图片的视觉结构（元素位置、层级、风格）
   - 结合文稿内容，生成 Remotion 组件和动画配置
   - 自动创建轨道、关键帧、动画效果
4. **视频预览**：
   - 实时渲染预览窗口展示效果
   - 用户可继续对话调整（"字体大一点"、"动画再快一些"）
5. **导出**：确认满意后导出视频或 Remotion 工程

## 对话调整
- 用户通过自然语言修改动画细节
- AI 定位对应元素并更新参数
- 支持撤销/重做

## 时间线编辑（可选）
- 可视化时间线显示所有轨道和关键帧
- 支持拖拽调整位置、时长、裁剪
- 点击元素可在属性面板手动微调
- 对话与手动编辑无缝切换，但不强制使用

## 素材导入
- 拖拽文件到项目面板或时间线
- 自动识别素材类型并分配到对应轨道

## 预览与导出
- 实时预览（Remotion 播放器）
- 导出视频（MP4/WEBM）或 Remotion 工程文件
- 支持导出预设配置（保存常用参数）
- **可选打开 Remotion 原生 Web 端**：有编程能力的用户可直接打开项目 Web 页面进行代码级修改
